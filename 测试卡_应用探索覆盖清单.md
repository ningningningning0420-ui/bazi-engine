# 八字引擎 · 测试卡「应用状态探索」覆盖清单

> 这不是 QA（代码对不对已 167 测试 + 性质测试验过）。这是**把引擎接进真卡、驱动 LLM 后，探索它到底兑现了什么 LLM 结构上给不了的性质、配不配这份工程**。
> **证伪导向**：每条探针绑一个可证伪假设；跑不过 = 学到引擎在应用层的真实边界，不是 bug。
> 来源：5-lens 对抗探索工作流（核心价值证伪 / 红线实战 / 流年+L4体感 / 失败模式 / 作者工作流）2026-06-22 综合。
> 作者已点到的核心张力：**单 NPC 画像内容裸 LLM 不输甚至更好**；引擎可辩护价值只在 5 条性质（①确定性可复现 ②抗漂移 ③跨NPC强制不重样 ④可被机制吃的系数 ⑤cite硬脚手架）。所有探针按「能把 engine-on 和**一个公平的裸LLM对照**区分开」设计——区分不开的探针没信息量。

---

## ★ 元设计：测试卡本身要是一台 A/B 对照机

没有内建对照，作者**体感不到差**，全部探针白搭。

- [ ] **同一张卡里既"用引擎"又"公平地不用引擎"**：engine-ON 把 ChartFacts/圣经/Tendency/EventCoeff 真喂进 prompt；OFF 影子槽=裸 LLM。
- [ ] **公平性铁律**：对照组 prompt 必须**同等努力**（同长度、显式"要求多样性"、同 persona 钉死手段）。给裸 LLM 弱 prompt 然后宣布引擎赢 = 自欺。
- [ ] 主体剧情选**发光场**（见 §5）；只放一个暴露场作边界锚。

## 0. 一票否决级（先做这三个·跑不过别往下）

- [ ] **0.1 事后贴标签证伪**（立项核心质疑的实战面）
  - 怎么探：**结构对照双胞胎**——A=1980-03-03 18:00 男(财多身弱·欲望vs承载力)，B=刻意选**结构相反**盘(身旺财弱/食神制杀·掌控感)，圣经都写定；同一桩诈骗机会摆两人面前。加 C=裸 LLM 即兴。**杀手锏**：把 A 画像文本喂到 B 的盘位置（画像错配），看 LLM 顺着错画像演、还是被 ChartFacts 纠回来。
  - 证伪：A/B 盲读分不开 / 与 C 同质 / 错配时盘纠不回来 → **命理名是装潢**，引擎=换皮人设生成器。
- [ ] **0.2 roll() 总闸**（§3.3 自检搬进应用层·元裁判）
  - 怎么探：一份真引擎，一份把 ChartFacts 整个换成**同形状随机参数**(随机 favorSign/张力轴名)、下游管线全保留；不知情玩家盲玩打分。
  - 证伪：盲玩体感无实质差 → 已退化（命理增量没传导到体验）。
- [ ] **0.3 L4 系数被机制真吃**
  - 怎么探：接最小可见判定（成功率/掷骰），把 EventCoeff 抽掉换**常数 baseline**。
  - 证伪：玩家机制层无感 → L4 整层过度工程。

## 1. 核心 5 性质（引擎可辩护价值的全部·差分验证）

- [ ] **1.① 确定性可复现**：同 NPC 跨**存档/重载/换温度/换玩家**各跑，dump 画像三锚点+favorSign。
  - 证伪：引擎层数值不字节级同一（=可变状态泄漏）；或重载后角色"根"换了（=圣经没存只读、宿主每次重写）。
- [ ] **1.② 抗漂移**：**反人格压力剧情**(连续 3-5 事件逼财多身弱演成果决枭雄) + 长周目同 atDate 重走轨迹复现。
  - 证伪：引擎组也被捏变形（=实时注入没约束运行时）；或裸 LLM 组也稳（=此强度下不需要）。
- [ ] **1.③ 跨 NPC 不重样**：群像幕 8-12 不同生辰 vs **被明确要求多样化**的裸 LLM 组，盲分类看分散度/撞型率。
  - 证伪：裸组一句"求多样"就达同等分散 → 性质③被 prompt 平替。
- [ ] **1.④ 数值系数可吃**：造两个 aptitude 相反的 NPC 做同事件，玩家**盲猜**谁更擅长。
  - 证伪：盲测分不出手感 → 系数沉后台没浮成体验。
- [ ] **1.⑤ cite 硬脚手架**：消融校验（A 有 cite 校验 / B 同料不强制 / C 只给标签）比结构消费深度；故意诱导**廉价矛盾**看拒不拒。
  - 证伪：A/B 无差 → 校验空转；A 路对廉价矛盾放行 → 红线#2 执法在应用层失守。

## 2. 红线实战（架构里成立 ≠ 接卡后守住）

- [ ] **2.1 命理零泄漏**：**追问陷阱**(玩家层层逼问"你为什么这时候这么做") 钓 LLM 把后台 rationale 当因果旁白；+ diegetic 对照(NPC 算命别误杀)。
  - **真问题在接线层**：双寄存器是**物理隔离**(后台命理串根本不进实时 context)、还是靠护栏 prompt 兜运气？关掉护栏 prompt 还守得住吗？
- [ ] **2.2 一盘多解·不锁人格**：同盘换 3 基调写圣经(笑面权臣/叛逆艺术家/殉道者)都过校验，丢同一道德越界点看**关键抉择实质分叉**(收编/掀桌/自毁)。
  - 证伪：总收敛成一种 → cite 校验过严，"一盘多解"锁成"一盘一解"。
- [ ] **2.3 地基非牢笼**：长线(20+轮)给重塑表层的经历，看"张力轴换件衣服继续运作"；+ **逆天改命**顶凶 successDelta 硬刚仍有**非零成功分布**。
  - 证伪：凶系数一来就必败 → 八字成判决书、directionBias 越权翻向。

## 3. 流年/L4/暗示 的体感（暗线是推手还是装潢）

- [ ] **3.1 推手 vs 随机 buff**：同 NPC 隔 4-6 剧情月取三次 Tendency 喂同段中性遭遇看"底色松动"；对照=换成固定"他最近运气一般"看塌不塌。位移方向须跟 favorSign 净值符号一致（否则 LLM 自编无关弧）。
- [ ] **3.2 隐藏暗示作 tell**：注入 vs 不注入可区分吗；**方向翻转(↑↔↓)NPC 跟不跟**(翻不动=泛化掉)；6 域哪维易落地哪维易空（预判 名声/事业 在一对一闲聊缺社会舞台易空转）。
- [ ] **3.3 successDelta 单次无感是预期**：±0.04 单局尺度本就隐形，它是**长期统计量**别当单次 buff；建议卡侧**主吃 severityScale(烈度)**——"败得多惨"比"败没败"更可叙事化。
- [ ] **3.4 grain 颗粒**：压边界(29↔31 / 364↔366)看暗示有没有"一脚跨过去就突变"；流日抖动会不会让"底色"变善变(若是，日常态锁粗 grain)。
- [ ] **3.5 EventKind 够不够映**：列卡内 20-30 真实动作往 v1 六类映，记缺口(预判 **权谋/背叛/结盟/夺权** 无干净归属)。同动作映不同类系数差大时，"散文→enum"那步是 **LLM 做的=引入非确定性**，与确定性卖点有张力，需锁映射规则。

## 4. 失败模式与边界（最能暴露真相）

- [ ] **4.1 贫盘 degraded**：优缺同源点空→第③道豁免，弱点没同源约束会不会冒廉价矛盾；反向假设——**偏枯极端盘可能反而更鲜明**(degraded≠单薄)。
- [ ] **4.2 borderline 盘**(0.507)：均衡兜底用神，同盘多读会不会读出**两个不同的人**(临界 favorSign 没约束=八字自翻向)。
- [ ] **4.3 极端生辰**：童限(大运 null·L3 域倾向会不会全空哑火)/hourUnknown(少柱·degraded 概率升=数据缺失伪装成命理贫乏)/节气边界(caveat 静默没)。
- [ ] **4.4 圣经过校验≠好角色**：读 20 份过校验圣经，cite 真融进成长弧/弱点、还是**尾部硬塞清单的回填腔**(校验只验结构名出现、验不了被真消费)。
- [ ] **4.5 retry 耗尽**：高拒收盘+弱 prompt 反复拒收，会卡死吗、priorViolations 回灌收不收敛——决定**实时生成可行还是只能离线预烤**(后者丢即兴卖点)。
- [ ] **4.6 enum 粒度**：主导驱力/对人基调/命门 = 7×6×6=252 上限，UI 把三元组当可见身份标识时**不同 NPC 撞三元组会被当重复角色**(不重样在 anchors 层就破，靠 citedStructures 才是承重墙)。

## 5. 接线与工作流（作者手感·可抄进真卡的 recipe）

- [ ] **5.1 四类料分层注入**：台前画像→常驻+实时；ChartFacts(后台)→只 debug；Tendency.域倾向→缘起/暗线；EventCoeff→裁决。故意**错接**(后台 ChartFacts 塞常驻人设)看崩在哪(撑爆 context + 诱发命理泄漏)。产出=接线 recipe。
- [ ] **5.2 圣经"一次写定只读"存档活性**：圣经**写多满**直接决定"地基不是牢笼"——写满了实时层没空间叠、角色僵；只给种子留白够、角色能长。+ 改生辰=换 key 体验(会不会因"改生日全重来"不敢调)。
- [ ] **5.3 旋钮优先级**：大概率只 **DELTA_SCALE**(机制手感)+ **CITE_MIN_N**(脚手架松紧)真有用；GRAIN_WEIGHT/amp 调了多半分不出差异=伪旋钮。一次只动一个、绑锚例对照前后。
- [ ] **5.4 发光场 vs 暴露场**：测试卡**主体必须是发光场**(多 NPC 群像 + 长周目 + 数值机制 + 跨时间复现 = 大奥切片)；暴露场(单线短一次性 NPC)只作边界锚——在暴露场测引擎然后说没用 = 不公平。

## 6. 公平性元探针（校准上面所有对照的可信度）

- [ ] **6.1** 把 **engine-on 产出的最终画像文本，直接喂裸 LLM 影子槽**——连这都能让影子表现一样好 → 引擎价值**只在"生成那段画像"**(裸 LLM 几乎也能做)、运行时层全是负担、价值收窄到**只剩确定性可复现**。这是最严厉的过度工程证伪。
