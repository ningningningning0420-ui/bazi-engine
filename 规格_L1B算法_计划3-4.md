## climate.ts — 调候（favor + pattern 共用单一出口）

`climateNeed(four, S) → ClimateResult`。`S=旺衰.五行得分`。
- **季节** `SEASON_BY_ZHI`：寅卯辰春 / 巳午未夏 / 申酉戌秋 / 亥子丑冬 / 辰戌丑未土月。先验 `CLIMATE_BASE_BY_ZHI`：亥子丑寒 / 巳午未暖 / 寅卯辰平偏湿 / 申酉戌平偏燥；辰丑湿 / 戌未燥。
- **量化**（读 S）：暖力=S.火；寒力=S.水；燥力=S.火+S.土×0.5；湿力=S.水+S.金×0.3+S.土×0.5。温度指数 T=暖力−寒力；湿度指数 H=湿力−燥力。
- 寒暖标签：冬月且 T<−`CLIMATE_T_GAP(0)`→偏寒，T<−`CLIMATE_T_STRONG(2.0)`→寒甚；夏月对称偏暖/暖甚。燥湿用 H 同法（标签 ∈ 寒甚/偏寒/平/偏暖/暖甚；燥甚/偏燥/平/偏湿/湿甚）。
- **调候提示**（0–2 个有序）：寒甚或(冬月且 S.火<`CLIMATE_FIRE_MIN(1.0)`)→['火']（寒甚补木→['火','木']）；暖甚或(夏月且 S.水<`CLIMATE_WATER_MIN(1.0)`)→['水']（暖甚补金→['水','金']）；燥甚→['水']；湿甚→['火','土']；土月辰丑→可['火']，戌未→可['水']再看 T/H；无偏颇→[]。
- **急迫度** `clamp01(max(|T|,|H|)/CLIMATE_T_STRONG)`。**级别**（单点产）：`急迫度>=CLIMATE_URGENT(0.6)?'critical':(调候提示.length>0?'adjust':null)`。`CLIMATE_URGENT` 只此一处声明。
- 输出 `ClimateResult{季节,寒暖标签,燥湿标签,温度指数,湿度指数,调候提示:WuXing[],急迫度,级别:'critical'|'adjust'|null}`。

## topology.ts — 涌现拓扑

`buildEmergentTopology(五行得分 S, 日主五行, 刑冲合害) → EmergentTopology`。只读 S（绝不重算）。
- **基线**：total=ΣS；share(w)=S[w]/total；mean=total/5。`isPresent=share>=PRESENT_SHARE(0.05)`；`isAbsent=S[w]<ABSENT_ABS(0.30)`；`isWeak=share<WEAK_SHARE(0.08)`。
- **流通度 flow**：active={isPresent}；n=|active|；flowEdges={w∈active|sheng(w)∈active}；linked={w∈active|sheng(w)∈active 或 shengMe(w)∈active}；coverage=|linked|/n；chainStrength=n<=1?0:|flowEdges|/n；`flow=round2(0.5×coverage+0.5×chainStrength)`；flowLevel：≥0.75畅/≥0.45通/≥0.2滞/else阻。breaks：无泄=isPresent(w)&&!isPresent(sheng(w))（v1 `EMIT_SOURCE_BREAK=false` 只报无泄）。
- **成环**：`isFullCycle = WU_XING.every(isPresent)`（生链是唯一5环）。longestChain：每个 shengMe(w)不在场的链头沿 sheng 走到尽头取最长 {path,len}。
- **相战对 clashes**：对 5 固定克对 (A,B) where ke(A)===B（木土/土水/水火/火金/金木）：bothStrong=share(A)>=`CLASH_MIN_SHARE(0.18)`&&share(B)>=同；balanced=min/max>=`CLASH_BALANCE_RATIO(0.5)`；势均=both&&balanced；通关神 X=sheng(A)（结构保证 sheng(X)===B）；hasTongGuan=isPresent(X)&&!isWeak(X)；**isClash=势均&&!hasTongGuan**；intensity=round2(势均度×(1−relief))，势均度=min/max，relief=isPresent(X)?clamp01(share(X)/share(弱方)):0。输出 `ClashPair{pair:[A,B],克:A,受:B,scores,tongGuanShen:X,tongGuanPresent,tongGuanScore,intensity,detail}`（pair 有序 ke(pair[0])===pair[1]）。clashes 只收 isClash===true。
- **通关清点 tongGuanReport**：全 5 克对登记 {forClash,node:sheng(A),present,score,relevant:势均(A,B),status:('调和中'|'缺位待补'|'无关'),detail}。
- **独旺 dominant**：share>=`DOMINANT_SHARE(0.40)` 或 share>=`DOMINANT_RATIO(2.2)`×其余四行均值。**独缺 missing**：isAbsent；附 类别（日主十神逆映射：同=比劫/shengMe=印/sheng=食伤/ke=财/keMe=官杀）。
- **交叉标注** `ANNOTATE_RELATION_CROSS=true`：clashes 两行若恰为某六冲/三刑涉及支本气五行，detail 追加"(地支亦冲)"——仅 detail，不改判定。
- 边界：total=0→空拓扑；纯读得分、无 tie-break、borderline 不影响。

## favor.ts — 用神 favorSign（★ 唯一真相源·100% 确定性）

`analyzeFavor(four, dm, strength, relations, topology, climate) → FavorResult`。**topology+climate 必传**。所有旺衰读 strength（不重算）。
- **角色五行**：dmW=dm.wuXing；yinW=shengMe(dmW)；bijieW=dmW；shishangW=sheng(dmW)；caiW=ke(dmW)；guanshaW=keMe(dmW)。same={yinW,bijieW}；drain={shishangW,caiW,guanshaW}。S=strength.五行得分；argmax/argmin 并列按 `WUXING_TIE_ORDER=['木','火','土','金','水']`。
- **§1 扶抑**：身弱→喜=same,忌=drain；身强→喜=drain,忌=same；均衡→喜=∅,忌=∅。扶抑主用神候选：身强=drain 三行 S 最高>0（并列 `FUYI_TIE_ORDER_身强=['官杀','财','食伤']` 类别映射后取首存在，全0→guanshaW）；身弱=same 两行 S 最高>0（并列 `FUYI_TIE_ORDER_身弱=['印','比劫']`，全0→yinW）。
- **§2 调候**：只读 climate.调候提示/级别；`tiaohou={需:climate.调候提示[0]??null,级别}`。只产喜不产忌。
- **§3 通关**：战对=topology.clashes；`tongguan={需:战对.map(c=>c.tongGuanShen),战对}`。通关 mid 入喜。
- **§4 汇总+冲突化解**：喜候选=Set(扶抑喜)+调候需+通关需；忌候选=Set(扶抑忌)。同一五行既喜又忌：调候 critical 行→从忌移除强制进喜；调候 adjust 行与忌冲突→降为闲(都移除,=0)；通关 mid 与忌冲突→保留进喜。
- **§5 主用神唯一决策树**（borderline 也唯一）：
  ```
  if 调候级别==='critical': 主用神=调候需, 来源='调候'
  elif 战对.length>0: pair=argmax 战对 by S[受](并列受 WUXING_TIE_ORDER); 主用神=pair.tongGuanShen, 来源='通关'
  elif 调候级别==='adjust': 主用神=调候需, 来源='调候'
  elif 身强弱!=='均衡': 主用神=§1扶抑主用神, 来源='扶抑'
  else balancedFallback()
  ```
  **balancedFallback**：maxW=argmax(S)（WUXING_TIE_ORDER）；主用神=keMe(maxW)（克最旺行）；喜加 shengMe(主用神)；忌加 maxW 与 shengMe(maxW)；来源='均衡兜底'。
- **§6 favorSign 赋值（−2..+2）**：全 0 → 喜候选=+1 → 主用神=+2（唯一）→ 忌候选中仍 0 者=−1 → pickChou()=−2。
  **pickChou**：候选=忌候选∩{favorSign≤0}。①keMe(主用神)∈候选→它=仇；②否则候选中 S 最高（并列 WUXING_TIE_ORDER）；③候选空→无仇。
- **§7 输出 FavorResult**：`{流派:'扶抑为主+调候补',主用神,来源,喜:WuXing[](favorSign>0 降序+2在前),忌:WuXing[](favorSign<0 升序-2在前),用神十神:TenGodCategory[](仅主用神一个类别·逆映射),favorSign:Record<WuXing,-2..2>,调候:{需,级别},通关:{需,战对:[{克,受,通关神}]}}`。
- **不变量（单测）**：恰一个 +2；−2 ∈{0,1} 个；均衡/borderline 绝不返空；同一五行无矛盾号；删 topology 参数即编译失败（必传）。

## combos.ts — 十神成立组合（计划 4·12 个·抗查表）

`detectGodCombos(facts) → ComboHit[]`（降序，过滤 `成立程度<MIN_DEGREE(0.1)`）。**degree=gate(硬前提)×Π(透藏×旺衰×相邻×制化)，无计数翻档平表。**
- **共享 helper**：`force(g)`=遍历四柱 present（日柱天干排除，藏干计入）累加（天干 STEM_WEIGHT；藏干 HIDDEN_WEIGHT[role]，月支藏干×MONTH_BRANCH_MULT）（三常量从 weights.ts import）。`categoryForce(cat)`；`fn(g)=clamp01(force/FORCE_REF(2.0))`。`transparencyFactor`：透根1.0/透无根0.7/藏本气0.5/藏中余0.3/无0（通根=该 god 所有承载干 any hasRoot）。`adjacencyFactor`：优先读 刑冲合害.贴邻→1.0；回退 SLOT_ORDER 差：贴邻1.0/同柱0.8/隔位0.5/纯藏0.4。`controllerPresence`：制化神 force 归一（CTRL_REF=2.0）。`clamp01`/`ramp(x,lo,hi)`。`bodyStrong01=clamp01((强弱比−WEAK_CUT)/(STRONG_CUT−WEAK_CUT))`；`bodyWeak01=1−bodyStrong01`。
- **12 组合 degree**（详见研究定稿，每个 gate + 连续乘积）：伤官见官(凶)/伤官配印(吉)/食神制杀(吉)/财滋弱杀(凶)/财多身弱(凶)/印重身旺(凶)/杀重无制(凶,gate f(七杀)>=1.5)/杀印相生(吉)/食伤生财(吉)/比劫夺财(凶)/官印相生(吉,须先算伤官见官缓存其 degree 做抑制)/伤官生财(吉)。**财多身弱/印重身旺 删身强弱标签 gate，改 bodyWeak01/bodyStrong01 连续门**（抗查表）。
- 输出 `ComboHit{类型,成立程度,极性:'吉向'|'凶向'|'中性',内含矛盾:boolean,members,力量快照,说明键}`。内含矛盾键集={伤官见官,财多身弱,杀重无制,印重身旺}。**成败清浊高低不 judge（红线6）**。
- 计算顺序：伤官见官 必先于 官印相生（抑制依赖）。

## tension.ts — 矛盾张力轴（计划 4·派生层，不重算成立度）

`buildTension(涌现拓扑, 十神组合, 日主五行) → TensionResult`。
- **X1 相战对派生**：遍历 clashes，两行各对日主十神类别查 `WUXING_PAIR_AXIS`（财×比劫=占有vs分享 / 官杀×食伤=权威vs才华 / 印×财=守成vs进取 / 比劫×官杀=自主vs受制 / 食伤×印=表达vs内省）；张力强度=clash.intensity；优缺同源=false。
- **X2 内含矛盾组合派生**（优缺同源核心）：读 内含矛盾===true 且 成立程度>`AXIS_MIN_DEGREE(0.3)`：伤官见官→'才华锋芒vs规则权威'[恃才放旷,守序服从]；财多身弱→'欲望vs承载力'[贪取扩张,力不能及]；杀重无制→'压力/野心vs失控'[进取担当,被压垮]；印重身旺→'庇护vs依赖'[厚学多受,迟滞被动]；优缺同源=true。良性组合一般不单列；反向共存(杀印相生+伤官见官)→派生'化解vs冲突'元张力轴。张力强度=成立程度×(有制化?`AXIS_RELIEF(0.6)`:1.0)。
- **X3 优缺同源点**：X2 优缺同源汇入 {结构cite,同一来源,正面表述,负面表述}（红线4 只定下限不锁人格）。
- **X4 去重排序**：同名轴合并取 max 强度，按强度降序，丢 `<AXIS_KEEP_MIN(0.2)`。
- 输出 `TensionResult{矛盾张力轴:TensionAxis[],优缺同源点[]}`；`TensionAxis{轴名,来源类型,来源cite,两极,张力强度,优缺同源}`。

## pattern.ts / cong.ts — 格局立格 + 从格信号（计划 4）

- **buildPattern(facts)**：monthGod=tenGod(日主.gan,月支本气 gan)；cat=category。建禄/月劫优先（cat===比劫→阴阳定建禄/月劫，羊刃=阳日&劫财）。八正格：透干天干(年/月/时，不含日)有本气十神透→立 `GE_NAME_BY_GOD[monthGod]`(本气透干)；本气不透→遍历中余气取首个透者(杂气透干)；全不透→格名+透干 false(月令本气暗藏未透)。多格信号→备选格[]。**不 judge 成格破格高低清浊**。输出 PatternResult。
- **buildCongSignal(facts)**（只信号不定论）：从弱=强弱比<=`CONG_WEAK_RATIO(0.10)`&&!得地.hasRoot&&!得令；从强=>=`CONG_STRONG_RATIO(0.90)`&&得令。细分触发项（从财/杀/儿/专旺倾向）。反例[]（极弱却有根/局中生扶日主之合化=假从/borderline→候选强制 null）。强度=clamp01(|r−0.5|×2×命中权重)；最终是否从格交 LLM。输出 CongSignal{候选,强度,触发项,反例}。
