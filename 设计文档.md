## 0. 一句话定位

**一台带因果约束的「人格语法」生成器 + 吉凶标度尺。** 从 NPC 生辰确定性地排出八字盘，把盘面的关系结构（而非性格结论）算深算准交给 LLM，让 LLM 在这套语法之上长出有厚度、自洽、不重样的人物；再用大运/流年/流月/流日作暗线推手随剧情时间推进，某些事件用流年做成功率 + 后果烈度/方向判定。

**它不是算命器。** 不追求预测准，只忠实地用命理系统的内部逻辑去生成深度——richness 来自系统本身。

---

## 1. 目的与要解决的问题

LLM 在剧情需要时即兴生成 NPC，常见两个病：

1. **薄背景板**：NPC 性格、经历单薄，撑不起需要的剧情深度与人物厚度。
2. **随剧情漂移**：NPC 没有自己的"势"，被当下剧情临时捏成需要的样子，前后不一致。

八字引擎的对策：给每个 NPC 一个**确定性、不重样、自洽**的人物底层（性格地基），并让它有一条随时间走的暗线（流年推手），使 NPC「自己有势在走」，而不是被剧情牵着鼻子。

### 1.1 为什么是八字，而不是「自己 roll 一组随机参数」

这是本项目的立项核心质疑，必须正面回答。结论：**只要引擎里出现任何「干支 → 性格词」的查找表，它就和随机参数 roll 同构、八字就白用了。本方案根本不做那张表。** 八字相对随机参数不可替代的价值落在三层，且这三层都在引擎里被算出来：

- **① 关系性**：随机参数是"外向度=0.7"这种孤立标量，每个值是孤岛、无"位置"概念。八字里没有任何一个字是独立的——同一个**七杀**，在「杀印相生」是贵气担当、在「财滋弱杀」是压垮日主的暴力、在「食神制杀」是被驯服的猛兽、在「杀重无制」是病灶。同一元素在不同关系网里是**完全相反**的人格信号。
- **② 涌现性**（方案硬通货）：核心运算是生克制化——**乘除与抵消，不是加减**。水克火，但有木通关，克就变成生。一张盘逐字都正常，可水火直接相战、中间无木通关 → 整盘涌现"撕裂感"。这个撕裂**不写在任何单字里**，是关系结构的拓扑性质，且**带因果地址**（"撕裂来自水火势均且无通关"）。随机能 roll"神经质=0.8"，但 roll 不出"他的神经质来自两股同样强的力量谁也压不倒谁"，更模拟不出"大运走木、撕裂暂缓"这种**底色随时间松动**的动态。
- **③ 内在自洽**：随机参数最大破绽是**矛盾廉价**（roll"既极度自律又极度放纵"零成本、读者立刻觉得假，因为没有"为什么"）。八字是**受约束生成系统**（五行守恒、生克链固定、十神由日主推导、旺衰由月令定），产出的矛盾自带因果：伤官见官的人又渴望规则认可、又忍不住挑衅权威——同一股伤官能量同时指向"表现欲"和"反建制"=**优缺同源**。约束系统产出"带解释的复杂"，随机拼盘产出"无解释的复杂"。

一句话：**随机参数给的是人格快照（一堆孤立标量），八字给的是人格语法（一套彼此约束、能生成这些标量的关系）。**

## 2. 范围与非目标

- **主消费方**：大奥这类卡——NPC 多、需要厚度、有真实 in-fiction 日期、React app（my-tavern-app 路线）。未来其它 React 卡复用。
- **核心形态**：**环境无关的纯 TS 逻辑库**（输入 datetime → 输出结构），再给各宿主接一层薄适配。对齐神隐 `lore.ts` 零依赖纯 TS 的复用模式。
- **独立性（可单独开启）**：插件是 per-卡 / per-session 的 **opt-in 开关**；虽为大奥而写，结构上必须能被干净拆出、供任何卡单用。所有大奥专属耦合只活在适配层，**核心零大奥依赖**（详见 §4.3）。关闭时核心不产出、不注入任何隐藏上下文，宿主行为如同插件不存在。
- **分层独立性**：**L0–L2（性格地基）只需生辰即可工作，任何卡可单用**；L3–L4（流年推手 / 成功率）需要宿主提供"当前剧情日期"，是**可选层**——宿主不提供时钟则只有性格地基生效。
- **非目标**：
  - **不**适配神隐论坛 / 审神者不妙屋（明确排除）。
  - **不**追求算命预测准确——只要系统内部逻辑自洽。
  - 核心**不**持状态、**不**掷骰、**不**调 LLM（这些归宿主/适配层）。

## 3. 核心哲学与红线

### 3.1 引擎 / LLM 分工原则

> **引擎只算「命理软件按公式必出同一值」的客观结构**（几何 / 查表 / 确定性演绎 / 确定性裁夺），把「这套结构是个什么人、这件事对他是吉是凶意味什么」的解读权全交 LLM。二者经**单一真相源**彻底切开：
> - **数学侧**（盘面几何 + 用神喜忌 + L3/L4 吉凶系数）100% 引擎确定，连 borderline 也用确定性 tie-break 强行定终值；
> - **叙事侧**（人物圣经 + borderline 的命理修辞）LLM 自由演绎，但被引擎结构**硬校验**锚定。
>
> 引擎 = 确定性的人格语法生成器 + 吉凶标度尺；LLM = 在这套语法之上长血肉的命师。两者经「用神锁 + cite 校验 + 真蝴蝶 gate」三道焊缝互不污染。

### 3.2 红线：八字是地基，不是牢笼

防止"算得深"做成"算死"、让八字变成"这个人只能这么演"的命理判决书。

- **① 结构定下限，不定上限。** 三道机器校验只拒两种东西——**违背结构**（把身弱断成铁血枭雄）和**空洞**（没真用上结构、廉价矛盾）。它们**不**要求"必须是这一种人"。cite 校验查"你用了没、有没有自相矛盾无因果"，**不是**"你给没给标准答案"。
- **② 一盘多解。** 同一个伤官见官——叛逆艺术家 / 腐败改革者 / 悲剧理想主义者 / 笑面权臣……**全都过校验**，各自尊重了"伤官见官"且自洽。引擎给的是**可多解的结构空间**，LLM 挑一条自洽读法，且**可为不同卡、不同基调读得不一样**。用神锁锁的是**数学侧吉凶计算**，不锁人格读法。
- **③ 地基之上还要生长。** 圣经是**性格地基/种子**，不是角色的全部、不是终点。角色照样在游玩里长——经历、关系、剧情事件层叠在地基之上，**表层行为可被重塑，根还在**。八字给的是根性/温度，不是每一句台词。
- **④ 流年是压力，不是剧本。** L3/L4 是暗线推力，角色可以**抵抗、无视、对着干**。系数只影响概率/烈度，怎么演交给 LLM。**逆天改命（顶着凶运硬刚）反而是好戏**——恰证明它不是唯一正确解。

一句话：**引擎保证 NPC 不薄、不自相矛盾，但不规定他是谁。**

### 3.3 防退化自检与红线清单

**自检口诀（总闸）**：把八字核换成 `roll()` 随机参数，系统行为有无实质变化？**无 = 已退化。** 挂在每个环节上。

**十条退化红线**（出现任一即"换皮"）：
1. 存在「干支 → 性格词」查找表
2. 只拿 LLM 预判标签、不拿盘
3. 盘只用一次（用完即弃）
4. 用神是配置常量（而非算法演绎）
5. 流年退化成随机 buff
6. 吉凶系数无 rationale
7. LLM 解读不受盘约束
8. 人格重算会漂移
9. 盘砍到只剩五行计数
10. LLM 边算边断（数学与叙事混流）

本方案对 ④⑦⑧ 三条做了针对性焊接（确定性工程用神 + cite 校验 + 算一次存档）。

### 3.4 输出纪律：命理零泄漏（正文不过度命理化）

八字是**纯后台机械**。引擎开启时，玩家应体验到一个有厚度、随时间变化的角色，但**正文里绝不出现引擎的命理推导**——不写"他这么做是因为天时/流年/命格/五行/旺衰/用神"。命理只是 LLM 的**内部依据**，不是叙事里说出口的因果。

这与 cite 校验（§5 L2）看似冲突，实则分属两个阶段、两个寄存器，解法 = **双寄存器**：

- **后台寄存器（命理推导）**：cite 具名结构、满足三道校验、持久化、可作大奥缘起的 latent-cause 依据。**永不作为叙事因果写进正文。**
- **台前寄存器（人物画像 / 行为暗示）**：用纯心理/行为语言表达的深度人格与当期倾向，**零命理词汇**。这才是注入实时生成上下文、驱动演绎的部分。

**深度不丢**：人物画像是 LLM 从命理结构"断"出来的，深度已烘进画像本身——命理术语不进实时上下文，角色照样厚。所有 **LLM 朝向的字符串**（圣经画像、L3 `隐藏暗示`/tell）一律命理-free 行为语言；命理 `依据`/`rationale` 只留后台。

**硬护栏（核心注入契约的一部分，每个适配层都必须带）**：
> 八字引擎开启时，正文严禁把命理推导当作叙事的解释/因果（天干地支、五行、十神、流年、命格、旺衰、用神等术语不得出现）。命理只是你的内部依据。

**唯一例外（剧情内命理 ≠ 引擎泄漏）**：角色在剧情里**主动**算命/谈命理（卦师、算命场景、本就信命的人物）是允许的——那是剧情内容，不是引擎把后台推导漏成旁白。护栏针对的是 **meta 泄漏**（把引擎的因果当旁白），不碰 diegetic 命理。

不做机械正文过滤器（金木水火土等常用字会误伤 + 误伤 diegetic 命理）；靠"注入台前寄存器 + 硬护栏 prompt"解决，可选留一个 debug 期 lint 仅作提醒。

## 4. 架构总览

### 4.1 引擎 / LLM 分工全景

| 层 | 引擎（确定性，birthInput 纯函数） | LLM |
|---|---|---|
| **L0 排盘** | 四柱 / 藏干 / 节气定月 / 连续纪日 | — |
| **L1 结构事实** | 十神全标注 · 通根 · 旺衰证据 · 刑冲合害定位 · **涌现拓扑**（流通/相战/通关/偏枯）· 十神成立组合 · 矛盾张力轴 · **用神喜忌**（含 borderline tie-break · 唯一真相源） | — |
| **L2 性格地基** | 给 L1 锚点 + 三道校验执法 | **写人物圣经**（一次性 · 存档 · 此后只读） |
| **L3 流年推手** | 引动倾向（域倾向 = 大奥缘起输入源格式）· grain 纯函数自动选 · 一句话隐藏暗示 | — |
| **L4 成功率** | 吉凶系数（successDelta + severityScale + directionBias）· **不掷骰** | 把散文事件裁成 EventKind enum |

掷骰权 100% 归大奥裁决引擎；八字只产系数喂进去，翻向仍由真蝴蝶 gate 焊死。

### 4.2 建议 core 库结构（复用 lore.ts 零依赖纯 TS）

```
bazi-core/
  constants/      # 干支、五行、藏干表、节气、十神定义、生克链
  chart/          # L0 排盘 + L1 几何（computeChart → ChartFacts）
  favor/          # analyzeFavor：确定性用神 + tie-break（唯一喜忌真相源）
  field/          # 力量场 + 关系算子 reducer
  forecast/       # selectGrain + L3 deriveTendency（流年/月/日 叠加）
  event/          # EVENT_FOCUS_TABLE + L4 eventModifier（固定最细 grain）
  epoch/          # worldEpoch + 五虎/五鼠遁 + 架空节气一致性检测
  rng/            # cyrb128 + mulberry32 + rollWithCoeff（standalone 兜底）
  validate/       # validatePersonaAnchors 三层校验
  *.test.ts       # 算子真值表 + 命例回归 + 复现性 + grain 边界 + 放大器叠加
```

### 4.3 Ports & Adapters（拆出独立性的承重设计）

为兑现"可单独拆出、普适"（决策 4），采用端口/适配器架构，**依赖方向单向：adapter → core，core 永不反向依赖任何宿主**。

- **core（bazi-core）**：纯函数 + 可持久形状定义。**零外部依赖、零大奥 import**（不碰 IndexedDB / 不碰大奥 RNG / 不碰缘起 / 不碰裁决）。
- **ports（宿主经参数注入，不被 core 反向 reach）**：
  - `seed/hash`：宿主提供 masterSeed 或 hash 函数（架空年映射、worldEpoch 用）。core 不自带全局 RNG。
  - `clock`（可选）：宿主提供"当前剧情日期"。**不提供 = L3/L4 关闭，只剩 L0–L2 性格地基。**
  - `persistence`（可选）：宿主提供读写 `{birthInput, 圣经/锚点}`。core 只定义形状，不假设存储介质。
  - `llmInvoke`（可选）：宿主提供"写圣经"的 LLM 调用；core 提供 prompt 契约 + `validatePersonaAnchors`，但**不自己调 LLM**。
- **adapters**：
  - **大奥-adapter**：域倾向 → 缘起、EventCoeff → 裁决、信封持久化、真蝴蝶 gate 对接（§8 全部内容属于此层）。
  - **generic-adapter**：把台前寄存器作隐藏 prompt 上下文注入 + 用 `rollWithCoeff` standalone 掷骰；给没有缘起/裁决的卡兜底。

## 5. 分层详细设计

### L0 排盘 · 地基

`computeChart(birthInput) → 四柱`。年柱（立春换年）/ 月柱（节气定月 + 五虎遁）/ 日柱（60 甲子连续纪日）/ 时柱（五鼠遁）。

**锁死 4 个 convention 随盘存档**：换年 = 立春 ／ 节气历版本 ／ 子时换日规则 ／ 时辰 = 钟表时。

**架空世界（决策 2 = 方案 a）**：`worldEpoch` 只映射**年柱**（hash 派生整年 → 落共轴）；要求生成侧给**真公历月日时**，月/日/时柱套真节气历 + 真连续纪日；`computeChart` 内做"映射年的节气边界 vs 给定公历月日"一致性检测，冲突 → 写 caveat 不静默。

**明文红线**：绝不对月/日/时柱做 hash 派生伪四柱（那会让月柱退化成无命理依据的随机查表 = 踩红线 #1 近亲）。

### L1 结构事实 · 引擎最高价值资产

全流派承认的盘面几何事实（零分歧或纯查表）：

- 藏干表 + 月令本气
- 十神全标注（各柱主气 + 藏干十神 / 透藏 / counts / categories）
- 通根明细（根支 + 根性质：本气/中气/余气/库根 + hasRoot）
- 月令旺相休囚死（纯查表）+ 得令得地得势（布尔与计数，**非打分**）
- 刑冲合害破会全关系检测（关系存在 + 位置 + 类型 + 贴邻，**不判吉凶**）
- 十二长生
- 神煞（**计划 7 已实现 · 单源本命神煞全集 · 38/38 闭合 · 忠实算**）：《三命通会》系四柱可确定性起例的本命神煞（38 · byStem/byYearBranch/byMonthBranch/byPillar/byNayin + 元辰 special），**纯描述结构料 · 进 cite · 不碰 favorSign**，吉凶/含义解读交 LLM。流派锁：三合类+孤辰寡宿年支起 · 火土同宫直接查表 · 羊刃只五阳干 · 魁罡含戊戌 · 天德合=合(天德) · 学堂词馆=年柱纳音长生/临官(土走水土同宫·作者拍板 A) · 天罗地网=纳音条件版(火命戌亥/水土命辰巳·男忌天罗女忌地网)。**纠正原“高戏剧低争议子集”的戏剧 curation**——按命理体系全集算，稀疏 present，LLM 定相关性。（deferred 5 项 2026-06-23 经 11-agent 多源研究+对抗校验补全·见 神煞_deferred研究_2026-06-23.md）

**★涌现拓扑（LLM 反推不出 · 整个方案硬通货）**：五行流通度 / 成环闭环 / 相战对（势均且无通关之克）/ 通关神有无 / 独旺行 / 独缺行——撕裂感/圆融/偏枯的**带因果地址**的结构来源。

**L1 深层锚点（引擎命名，非裸字）**：

- 十神成立组合（伤官配印 / 伤官见官 / 食神制杀 / 财滋弱杀 / 财多身弱 / 印重身旺 / 杀重无制…各附**连续成立程度 0–1**，抗查表最关键项）
- 矛盾张力轴（相战对 + 内含矛盾组合 + 优缺同源点 汇成几条内在对立轴）
- 旺衰判定走 borderline 带（引擎只出全部证据 + `borderline:true`，定性留叙事侧）
- 格局立格（月令本气透干机械识别）+ 从格极端信号；成败高低清浊**留叙事侧**
- 调候（寒暖燥湿）

**★用神 · 唯一权威喜忌真相源 · 100% 引擎确定（决策 1 = A）**：`analyzeFavor` 用确定性扶抑算法（帮扶方计数 + 位置权重 vs 耗克方计数）+ 调候硬规则；**即便旺衰 borderline，也用固定 tie-break 优先级（调候 > 通关 > 扶抑）强行算出唯一** favorableElements / unfavorableElements / 用神十神，对 LLM 完全不可改。`favorSign[木火土金水] ∈ {+2 用, +1 喜, 0 闲, -1 忌, -2 仇}` 由此**演绎**（绝非配置常量 · 防退化 #4）。L3/L4 一切吉凶**只读它、锁死**。
代价：工程用神在真临界盘上可能"不那么命理正确"，但作者明确不在乎算准——确定性 tie-break 本身就是系统内部逻辑自洽，可接受。
v1 硬选**扶抑为主 + 调候补**；多流派（扶抑/调候/通关/病药）作可配置策略，但**一局一锁**（开局选定 + masterSeed 锁全程一致），流派配置留 v2。

### L2 性格地基 · LLM 写圣经 + 引擎三道校验

> ★**双寄存器（命理零泄漏，详 §3.4）**：圣经分「后台命理推导」（cite 结构 / 过校验 / 持久化，永不外露）与「台前人物画像」（纯行为语言 / 零命理词 / 注入实时上下文）。下述 cite 校验作用于**后台寄存器**；实时演绎只读**台前寄存器**。

LLM 只做两件事，皆被引擎结构硬校验锚定：

**写人物圣经（读 L1 锚点）**：像命师把"工程身强弱 + 引擎用神 = 水木 + 伤官见官 + 财星透干无根"这堆关系性结论**断成一个人**：性格地基 / 动机 / 说话方式 / 关系模式 / 弱点 / 成长弧 / 阿喀琉斯之踵。**一次性写定存档，此后只读**（可复现）。borderline 的命理修辞（身强偏弱、合化成立与否、格局成败清浊）由 LLM 在圣经**叙事文本**里随意说（"命师认为他身强偏弱、念旧"），但**绝不回写 favorSign / 喜忌符号**——叙事用神与数学用神彻底分家。引擎里根本不存在"干支→性格词"映射，人格永远 LLM 当场长出。

**三道机器校验**（`validatePersonaAnchors`，违规拒收重写）：

1. **子集校验**：滋养向必须 ⊆ favorableElements；身弱用印不许断成铁血枭雄（那是身旺财杀断法）。
2. **★cite 强制校验**：圣经长文必须**显式 cite ≥N 条 ChartFacts 具名结构**（十神成立组合名 / 矛盾张力轴 / 相战对 / 通关神有无）；引擎校验"cite 的结构名 ∈ ChartFacts 实际存在的结构"，且"每条矛盾张力轴都在圣经的成长弧或弱点里被消费"。cite 不满或 cite 了盘上不存在的结构 → 拒收。逼 LLM 真用上涌现拓扑，而非回填几个合法 enum 交差。
3. **★矛盾同源校验**：圣经的"弱点/阿喀琉斯之踵"必须与"主导驱力/优点"挂在**同一个十神组合 cite 锚**上，挡掉廉价矛盾。

回填两层 = 长文圣经（叙事软态给 LLM 读）+ 枚举锚点（L1 持久给引擎校验/后续引用）= 大奥"储君倾向 dial 账本 + 三观软态裁判养"同构。

**正交补强（L4 × L2 不矛盾反成戏剧张力）**：用神受克的流年，L4 走数学判"此事客观易败"，L2 软态演"但他偏偏此时格外执拗"——数字管成败烈度、剧情管怎么演。reducer 落账只认 L4 数学系数夹幅度，L2 解读只进演绎散文。

### L3 流年推手 · 复用同一确定性内核

`deriveTendency(chart, atDate, grain) → Tendency`。把流运柱当新干支，复用十神标注 + 刑冲合化检测；大运排布（起运岁 + 顺逆按阴阳男女）、当运干支、作用[]、域倾向[]（事业权位 / 财 / 关系情感 / 健康 / 心性波动 / 名声口舌 + 方向 ↑↓动荡平 + 命理依据）、一句话隐藏暗示。

- **grain 确定性**：`selectGrain(timeSpanInFictionDays)` 纯函数定，阈值硬编码（≥365 → 流年；30..365 → 年+月；<30 → 年+月+日）；输入**只认 in-fiction 日期跨度**，禁用"剧情节奏感"。
- **关键解耦**：grain 只决定 L3 文本暗示颗粒度，**不决定 L4 数值**。
- **喂大奥缘起**：`域倾向` 直接是大奥缘起一路输入源格式；L3 暗示可 arm 成 latent-cause(H)。
- **命理零泄漏（详 §3.4）**：`域倾向.依据` / `rationale` 是后台命理（喂缘起 / debug）；`隐藏暗示` 是台前**行为语言、零命理词**（喂 tell 池）。

### L4 成功率 · 纯几何 · 不掷骰

`eventModifier(chart, atDate, eventKind) → EventCoeff`。用神已确定故 L4 纯几何。

- **L4 永远用最细 grain（年+月+日）算 timing**：L4 数值与 grain 完全解耦。
- `EVENT_FOCUS_TABLE` 把 eventKind 映到关注十神/五行/宫位。
- 公式 = `clamp(0.4×命局禀赋 aptitude + 0.6×当期引动 timing, -1, 1) × 放大器 amp`，输出 `{successDelta, severityScale, directionBias, rationale}`。
- **放大器叠加锁**：多放大器**取 max 不连乘**（若连乘则 severityScale 硬 clamp 到 2.0）；反吟/伏吟/岁运并临的比较基准集锁死 = 对**原局四柱 + 当运大运**（不含流月流日，消除 grain 依赖）；全进算子真值表单测。
- **掷骰权 100% 外置**；核另给 `rollWithCoeff(coeff, prng)` standalone 兜底（通用卡用，大奥消费方不调）。

## 6. 确定性与可复现

### 6.1 用神 100% 引擎确定（决策 1 = A，承重墙修补）

校验阶段捅穿的承重墙裂缝：原构想"引擎算用神、但 borderline 终判交 LLM"自相矛盾——L3/L4 全部吉凶吊在用神这根柱子上，而它一旦交 LLM，重 roll 圣经/换温度会让用神漂移 → 不可复现（踩红线 #4 + #8）。自检口诀给尴尬答案：把"LLM 裁定 borderline 用神"换成 `roll()` 在候选里选一个，下游 L3/L4 无实质变化 = 该步已退化。

**修法（方案 A，最高杠杆单点改动）**：数学用神 100% 引擎确定（含 borderline tie-break），LLM 完全不可改、L3/L4 只读它；LLM 的 borderline 终判只进圣经叙事文本、**绝不回写 favorSign**。这才真正兑现"拆两角色让冲突消失"。结果：ChartFacts 重新成为 birthInput 的**纯函数**，"比大奥资质 roll 更纯、连 RNG 都不需要"为真。

### 6.2 持久化策略

- 八字盘是 birthInput 的确定性纯函数 → **ChartFacts 只读不持久**（载入时重算，像 canonRef → findSwordLore）。
- **只持久化** `birthInput + LLM 圣经 / 锚点`（存档体量极小）。
- 改 NPC 生辰 = 换 key = 真重排盘（诚实边界）。

### 6.3 RNG（仅两处，全 hash 派生稳定坐标，无可变计数器）

1. 架空整年映射 = `hash(masterSeed ⊕ npcId ⊕ "birthYear")`
2. `worldEpoch` 一次性派生 `{anchorStoryYear, anchorGanzhiIndex}` 存信封 meta 不可变 = 全架空世界共用一条 60 甲子纪年轴（同年集体犯太岁可呼应）

### 6.4 确定性吉凶数学骨架

力量场 × 关系算子 × 喜忌符号 三段纯函数，替代死查找表：

- 吉凶净值 = `Σ(场扰动[五行] × favorSign[五行])` 内积。
- `favorSign` 是指南针、方向完全由它提供——同一个"冲"作用喜神 = 凶、作用忌神 = 吉，**绝不写死**。
- 关系算子（生克 / 五合 / 六合 / 三合三会 / 六冲 / 三刑 / 六害 / 墓库冲开 / 岁运并临 / 反吟 / 伏吟）各自只改力量场 + 吐机器谓词 enum，按强度优先级 reducer 结算（会 > 合化 > 三合 > 冲 > 合绊 > 刑 > 害 > 生克，贪合忘冲可单测），**不直接判吉凶**。
- 强度档 0–3 映射与大奥 Δ 档（无/轻/中/重）同构。

### 6.5 可复现单测纪律（黄金样例三类）

1. **算子真值表**：关系算子 + 放大器叠加边界做成 golden case（含取 max 不连乘 / 比较基准集只含原局+大运）。
2. **已知命例回归**。
3. **复现性**：同 birthInput 两次 computeChart 字节级等同；grain 边界 ±1 天净吉凶连续性不变量；L4 数值 grain 无关性断言。

## 8. 大奥适配层（adapter · 非核心）

> 以下全部属于 **大奥-adapter**，核心零依赖、零反向 import；换一张卡换一个 adapter（见 §4.3）。

- **zod schema-first**：零额外约定（= 写 TS 类型 = 写 zod schema）。
- **Verdict ≠ StatePatch 单 reducer**：ChartFacts 绝不直接写 GameState（只读派生）；NPC 圣经/锚点经 LLM 圣经 call 走 reducer 落档案（像 verdict 落账校验+落）；L3/L4 系数只作 verdict 输入材料，最终 Δ 仍由裁判吐 verdict、reducer 夹落账——**八字不绕单写口**。
- **IndexedDB 信封 {meta, state} migrate-on-load**：只持久化 birthInput + 圣经/锚点，ChartFacts 载入重算。
- **缘起暗线**：`deriveTendency().域倾向` 直接是缘起输入源格式；L3 暗示可 arm 成 latent-cause(H)。
- **真蝴蝶 gate**：八字产 directionBias / severityScale 喂裁决，**翻向归 §14.1 焊死 + per-key 方向 gate**；八字流年**不自己翻向**。successDelta 只是 baseRate 修正量喂大奥裁决引擎，绝对幅度走 DELTA_SCALE 校准旋钮。
